# 天球网格与地平网格间隔调整计划

## 背景
当前星图中的地平网格（方位/高度）与天球网格（赤经/赤纬）采用较密的 15° 间隔，视觉上偏拥挤，影响阅读。

## 目标
将两类网格的主间隔增加一倍（15° → 30°），降低线条密度并保持网格结构一致性。

## 实施方案
1. 在 `lib/widgets/star_chart.dart` 的网格绘制逻辑中，将以下循环步进由 `15` 调整为 `30`：
   - 地平网格：高度线、方位线。
   - 天球网格：赤纬线、赤经线。
2. 为避免 30° 步进导致关键 `0°` 线缺失，调整赤纬/高度循环起止范围为 `-60..60`，确保包含 `0°` 主线并维持均匀分布。

## 验证
- 运行相关测试（至少包含星图投影测试）确保改动未破坏现有行为。

## 风险与回滚
- 风险：边缘纬线数量减少，极区参考线显示略少。
- 回滚：将步进恢复为 `15`，并还原循环范围即可。
