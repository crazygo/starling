# Plan: 巡天视图限制为水平旋转，降低眩晕与“圆周绕点”感

## 背景
用户反馈在巡天（dome）视图拖动时，容易出现“整个屏幕围绕某个点旋转”的感受，导致眩晕。

当前交互中，dome 模式会同时响应水平与垂直拖动：
- 水平拖动改变方位角（azimuth / RA）
- 垂直拖动改变高度角（altitude / Dec）

这在视觉上等价于同时存在 yaw + pitch，相比“只左右看”的预期更容易产生空间失配。

## 目标
1. 将 dome 模式的手势平移约束为“仅水平旋转”（绕纵轴）。
2. 保持 classic 模式原有水平+垂直平移行为不变。
3. 鼠标/触控板滚动平移逻辑与触摸拖动一致（dome 下也仅水平）。
4. 增加测试，确保该行为可回归验证。

## 实施方案
1. 在 `lib/widgets/star_chart.dart` 中抽出一个可测试的垂直平移计算函数：
   - dome：返回原始 `centerDec`（不因平移手势而变化）
   - classic：沿用现有 `deltaY -> Dec` 的线性映射并做边界钳制
2. 在 `_onScaleUpdate` 与 `_onPointerSignal` 中统一调用该函数，避免逻辑分叉。
3. 在 `test/star_chart_projection_test.dart` 增加单元测试覆盖：
   - dome 下垂直拖动不改变 Dec
   - classic 下垂直拖动仍会改变 Dec

## 影响评估
- 正向：显著减少“绕屏幕某点旋转”的体感，用户更容易获得“左右扫视”体验。
- 兼容：classic 视图不受影响。
- 取舍：dome 手势将失去手动上下看能力；如需上下看，后续可通过陀螺仪或单独开关恢复。

## 验收标准
- dome 模式下，任意上下拖动/滚动不会改变中心高度（Dec）。
- dome 模式下，左右拖动仍可稳定改变方位。
- classic 模式下，拖动与滚动平移行为与改动前一致。
- 新增测试通过。
