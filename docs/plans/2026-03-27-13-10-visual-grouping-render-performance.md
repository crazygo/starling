# 视觉分组与渲染性能优化计划

## 背景
用户希望优化星图渲染性能，并新增“视觉分组”设置：
1. 显示非属于星座/星官的星星（默认关闭）
2. 仅对主要星星显示标签（默认开启）
3. 背景星星阈值（小/中/大，单选），用于“星星实际渲染大小低于阈值时不显示星名”

同时要求：
- 星座与星官名称始终显示。
- 评估当前是否只渲染屏幕范围内星星与计算量。

## 现状分析
- 当前 `_project` / `_projectStar` 会在经典投影中进行视口裁剪；穹顶投影中基于相机深度裁剪，仍可能对屏幕外点做一定计算。
- 星体绘制与标签计算会遍历全部目录星（约 8870 颗），并依赖可见性判断与亮度阈值进行筛选。
- 目前缺少用户可配置的“视觉分组”策略。

## 实施方案
1. **设置模型扩展**
   - 在 `SettingsService` 新增：
     - `showNonConstellationStars`（bool，默认 false）
     - `majorStarsOnlyLabels`（bool，默认 true）
     - `backgroundStarThreshold`（enum: small/medium/large，默认 medium）
   - 加入 SharedPreferences 持久化与读取。

2. **设置页新增视觉分组分区**
   - 在 `SettingsPage` 增加“视觉分组”section，包含两个开关与一个单选组。

3. **星图渲染策略接入**
   - `StarChart`/`_StarPainter` 接收上述三项设置。
   - 计算“标签最小渲染半径阈值”映射：
     - 小=0.8，中=1.4，大=2.0（可后续微调）。
   - 星名标签规则：若星体实际渲染半径 `< 阈值`，不显示星名；星座/星官名称不受影响，始终显示。
   - 当“仅对主要星星显示标签”为真时，仅保留 Group1/Group2 + 高亮主星（按较亮星等）标签，抑制竞争标签。
   - 当“显示非属于星座/星官的星星”为假时，绘制层跳过非成员星。

4. **渲染性能小优化**
   - 引入屏幕边界快速裁剪（含小 margin），减少屏外绘制调用。
   - 统一缓存星体屏幕坐标与半径，供绘制与标签共用，减少重复投影/计算。

5. **验证**
   - 运行格式化与静态检查/测试（至少 `flutter test`）。
   - 自检设置默认值、切换行为、标签与绘制策略一致性。

## 风险与回滚
- 风险：标签密度变化可能影响可读性。
- 缓解：通过阈值档位与默认值（中）平衡；星座/星官名始终显示兜底。
- 回滚：可仅回退视觉分组策略，不影响核心投影与数据加载。
