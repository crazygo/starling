# 环境初始化与变更验证补充计划

## 背景
上一次提交中，`flutter` / `dart` 命令在执行环境缺失，导致未能完成格式化、l10n 生成与测试验证。
本次要求使用仓库提供的初始化脚本安装缺失命令并完成验证。

## 目标
1. 使用 `scripts/init_codex_env.sh` 初始化 Codex 环境，补齐 Flutter/Dart。
2. 在同一会话中执行并验证关键命令：
   - `flutter gen-l10n`
   - `dart format`（针对改动文件）
   - 测试命令（至少 `flutter test`）
3. 如验证导致代码变更（格式化、自动生成），将变更纳入提交。

## 步骤
1. 执行初始化脚本并确认 Flutter/Dart 版本可用。
2. 运行 l10n 生成与格式化。
3. 运行测试并修复发现的问题。
4. 提交修复并同步 PR 信息。

## 风险
- 初始化过程依赖网络下载，可能失败或耗时较长。
- 新版本 Flutter 可能触发额外格式化/生成差异，需要一并提交。
