# Plan: 巡天页面恢复上下滑动，并在单次手势中做轴向锁定

## 背景
此前为了降低眩晕感，巡天（dome）视图将垂直拖动完全禁用，只允许水平旋转。当前用户反馈需要恢复上下滑动能力。

同时，若允许完全自由的二维拖动（水平+垂直同时生效），体验会变差，容易出现方向感漂移。用户希望在“触发上下滑动”时保持地平经线始终竖直。

## 目标
1. 巡天视图重新支持上下拖动/滚动（调整中心高度）。
2. 巡天视图避免“全自由”二维平移：单次手势按主轴锁定（水平或垂直）。
3. classic 视图交互行为保持不变。
4. 增加/更新测试，覆盖 dome 垂直平移与轴向判定。

## 实施方案
1. 在 `lib/widgets/star_chart.dart` 增加可测试的 dome 主轴判定函数：
   - 引入轻微 dead-zone，防止手势起始抖动误判。
   - 超出 dead-zone 后以 `|dx|` 与 `|dy|` 比较确定主轴。
2. 在 `State` 中记录 dome 当前手势锁定轴：
   - `onScaleStart` 清空轴。
   - `onScaleUpdate` 首次超过 dead-zone 后锁定为水平或垂直，并在整个手势内只应用该轴位移。
3. 调整 dome 垂直平移逻辑：
   - `panCenterDecForStyle` 在 dome 下不再固定不变，而是根据 `deltaDy` 更新并做高度钳制。
4. 对滚轮/触控板滚动同样采用“按事件主轴”过滤，保证与触摸拖动风格一致。
5. 更新 `test/star_chart_projection_test.dart`：
   - dome 下垂直平移会改变 Dec（并被钳制）。
   - 新增 dome 主轴判定测试。

## 验收标准
- 巡天视图可通过垂直手势上下看天。
- 巡天视图中单次手势不会同时水平+垂直自由漂移（有主轴锁定）。
- classic 视图行为不受影响。
- 相关测试通过。
