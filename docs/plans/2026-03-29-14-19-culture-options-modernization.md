# 文化选项升级计划（中国古代/中国现代/西方）

## 背景
当前文化选项只有“中国文化”和“西方文化”。
需求是：
1. 将“中国文化”重命名为“中国古代（步天歌）”；
2. 新增“中国现代”；
3. 同步完成数据下载与数据编译链路改造，确保功能端到端可用。

## 目标
- 设置页展示三种文化选项：
  - 中国古代（步天歌）
  - 中国现代
  - 西方文化
- 运行时渲染策略：
  - 中国古代：使用中国古代星官连线 + 中文星名；
  - 中国现代：使用现代 88 星座连线 + 中文名（星名/星座名）；
  - 西方文化：使用现代 88 星座连线 + 英文名。
- 数据管线新增 `culture_chinese_modern.bin` 产物，并接入下载/生成脚本。

## 实施步骤
1. 扩展 `CultureMode` 枚举和设置持久化兼容逻辑。
2. 增加本地化文案 key：`chineseAncientCulture`、`chineseModernCulture`。
3. 设置页改为三选项展示。
4. `StarDataService` 增加中国现代文化 bin 加载与解析。
5. `ExplorePage` 按文化模式分别决定：名称语言与连线数据。
6. 工具链改造：
   - pipeline 生成 `culture_chinese_modern.bin`；
   - generate_bins 校验新增产物；
   - README / schema 注释更新；
   - 下载脚本注释与流程同步。
7. 更新/新增测试，验证三模式行为与新 bin 生成。

## 验证
- 运行相关单测（至少覆盖设置页文案、数据加载、模式映射逻辑）。
- 运行工具链测试或最小命令验证 pipeline 可输出四个 bin（含新文件）。

## 风险与回滚
- 风险：新增 bin 未随 assets 一起打包导致运行时加载失败。
  - 应对：确保 `pubspec.yaml` 使用目录打包策略；单测覆盖加载流程。
- 风险：历史配置值迁移不兼容。
  - 应对：`load()` 中保留旧值兼容和默认回退。
